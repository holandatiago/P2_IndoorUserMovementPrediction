Para resolver o problema eu imaginei que, como as trajetórias não se cruzam, os pontos médios de cada uma seriam bastante distintos. Então para cada um  dos 314 conjuntos de dados, eu calculei as médias dos RSS de cada âncora. A partir daí eu usei uma árvore de decisão binária para fazer a classificação, por ser um algoritmo simples, eficiente e o conjunto de dados provavelmente não ser linearmente separável. Essa árvore foi importada da biblioteca scikit-learn.
Como os dados de entrada estão ordenados por ambiente e trajetória, resolvi dividí-los em ímpares para o conjunto de treinamento e pares para o de teste. Assim eu consigo garantir que em cada conjunto exista uma quantidade representativa de dados para cada configuração de entrada.
Após o treinamento, calculei o índice AUC para o conjunto de teste, que deu aproximadamente 0,85. Cheguei à conclusão de que apesar do meu método ser bastante simples, ele obteve um resultado suficientemente satisfatório.
